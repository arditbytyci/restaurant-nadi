## Issues Found

### ЁЯФ┤ Critical тАФ About page has its own DIY smooth scroll fighting Lenis

`app/about/page.tsx` implements a **completely custom `requestAnimationFrame` scroll loop** with lerp (`* 0.04`) on a `fixed` container. This means you have:
- Lenis running globally
- A custom scroll loop running on top of it
- Both fighting each other тЖТ major jank

The fix is to ditch the entire custom scroll logic and let Lenis handle it. The about page also has a `<style jsx>` block (requires `styled-jsx` which isn't installed) and `WebkitOverflowScrolling` which is deprecated.


Every time you scroll back up and then down, it re-animates the section with `y: 50 тЖТ 0`. Change to `once: true`.

**Component file anatomy (what's professional inside a file):**
```tsx
// 1. Imports тАФ external first, then internal
import { useState } from "react"
import { motion } from "framer-motion"
import { Container } from "@/components/layout/Container"

// 2. Types/interfaces above the component
interface HeroProps {
  title: string
  subtitle?: string
}

// 3. Constants outside the component (not inside тАФ avoids re-creation)
const ANIMATION_VARIANTS = { ... }

// 4. Named export (not default тАФ easier to refactor/find)
export const Hero = ({ title, subtitle }: HeroProps) => {
  return (...)
}
```

**Export style тАФ named over default:**
```tsx
// тЭМ default export тАФ harder to track, can be renamed on import
export default function Hero() {}

// тЬЕ named export тАФ explicit, refactor-safe, autocomplete-friendly
export const Hero = () => {}
```

Your project mixes both. `SliderComponent`, `MenuPage`, `AboutPage` are default exports while most components are named. Pick named exports and stick to it тАФ the only exception is Next.js pages/layouts which *require* default exports.

**Barrel files (`index.ts`) done right:**
```ts
// components/icons/index.ts
export { Facebook } from "./Facebook"
export { Instagram } from "./Instagram"
export { Location } from "./Location"

// Then import anywhere as:
import { Facebook, Instagram } from "@/components/icons"
```

**Hook naming:**
```ts
// тЭМ what you have тАФ logic buried in components
const [isMobile, setIsMobile] = useState(false)
useEffect(() => { ... window resize ... }, [])  // repeated in 3 files

// тЬЕ extract to a hook
// hooks/useMediaQuery.ts
export const useIsMobile = () => {
  const [isMobile, setIsMobile] = useState(false)
  useEffect(() => { ... }, [])
  return isMobile
}
```

You repeat the mobile detection pattern in `about/page.tsx`, `scroll.tsx`, and `Header/index.tsx` тАФ that's a textbook case for a shared hook.
